Daily Draw Page
-----------------------------
၁။ Minimal Header (အပေါ်ဆုံးပိုင်း)
 - Title/Logo: အလယ်မှာ App ရဲ့ နာမည် သို့မဟုတ် Logo အသေးလေး ပါပါမယ်။

၂။ Instruction Area (လမ်းညွှန်ပိုင်း)
- Title: "ယနေ့အတွက် သင်၏ ဟောစာတမ်း" (သို့) "Card of the Day"
- Subtitle: "စိတ်ကို တည်ငြိမ်အောင်ထားပါ။ အဆင်သင့်ဖြစ်လျှင် ကတ်ကို နှိပ်ပါ။" (အသက်ရှူလေ့ကျင့်ခန်း လုပ်သလိုမျိုး ဖြည်းဖြည်းချင်း မှေးမှိန်ပြီး ပေါ်လာတဲ့ Fade-in Text Animation လေး သုံးလို့ရပါတယ်)။

၃။ The Interactive Stage (အဓိက ကတ်ပြသမည့် နေရာ)
- မျက်နှာပြင် အလယ်တည့်တည့်မှာ ကတ်တစ်ကတ်တည်းကို မှောက်လျက် (Card Back) ပြထားပါမယ်။
- Pre-click (မနှိပ်ခင်): ကတ်လေးက လေထဲမှာ ပေါလောပေါ်နေသလိုမျိုး အပေါ်အောက် ဖြည်းဖြည်းချင်း ရွေ့နေတဲ့ Animation (Floating Effect) လေး သုံးထားပါမယ်။ Mouse တင်လိုက်ရင် (Hover) ကတ်ရဲ့ ဘေးပတ်လည်မှာ အလင်းရောင် (Glow) လေး ထွက်လာပါမယ်။
- The Click (နှိပ်လိုက်ချိန်): ကတ်ကို နှိပ်လိုက်တာနဲ့ CSS 3D Transform ကို သုံးပြီး Y-axis အတိုင်း ချောချောမွေ့မွေ့ လှန်သွားပါမယ် (Flip Animation)။

၄။ Result Area (ရလဒ်ပြသမည့် နေရာ - ကတ်လှန်ပြီးမှ ပေါ်လာမည်)
ကတ်လှန်သွားတာနဲ့ တစ်ပြိုင်နက် ကတ်ရဲ့ အောက်ဘက်မှာ အောက်ပါ အချက်အလက်တွေက Fade-in နဲ့ အသာလေး ပေါ်လာပါမယ်။
 - ကတ်နာမည်: ဥပမာ - The Fool
 - အဓိက စကားလုံးများ (Keywords): စွန့်စားခြင်း၊ အစပြုခြင်း၊ လွတ်လပ်ခြင်း။
 - တစ်နေ့တာ ဟောစာတမ်း အတို

၅။ Next Action (နောက်ထပ် လုပ်ဆောင်ရန် ခလုတ်များ)
- Action Button: * မှတ်စုတွင် သိမ်းရန် (Save to Journal) ခလုတ်ကို ဟောစာတမ်းအောက်မှာ အလယ်ဗဟိုကနေ ထင်ထင်ရှားရှား ပြထားပါမယ်။
  * Logic: နှိပ်လိုက်ရင် JavaScript က auth.js ထဲက Token ကို စစ်မယ်။ Token မရှိဘူး (Guest) ဆိုရင် /login ကို သွားဖို့ သတိပေးချက် (Modal/Alert) လေး ပြပြီး လမ်းကြောင်းလွှဲမယ်။ Token ရှိတယ် (Member) ဆိုရင် Supabase ကို လှမ်းပို့ပြီး သိမ်းမယ်။

- အောက်ဆုံး Navigation (ဘယ် / ညာ ခွဲထားခြင်း):
  * ဘယ်ဘက် (Left): <- နောက်သို့ (Back) (Home Page သို့ ပြန်သွားရန်)
  * ညာဘက် (Right): အခြား ဟောစာတမ်း ဖတ်ရန် -> (New Reading) (/reading Page သို့ သွားရန်)

** နောက်ကွယ်က Logic အလုပ်လုပ်မည့်ပုံ (Developer View)
အဆင့် (၁) - User အခြေအနေကို စစ်ဆေးခြင်း (Check Auth State)
User က ဒီ Page ကို ရောက်တာနဲ့ script.js ကနေ Browser ထဲမှာ အကောင့်ဝင်ထားတဲ့ Token ရှိ/မရှိ အရင်ဆုံး စစ်ဆေးပါမယ်။

အဆင့် (၂) - Member ဖြစ်ခဲ့လျှင် (Token ရှိလျှင်)
- Vercel ကနေတစ်ဆင့် Supabase Database ဆီကို ဒီနေ့ ရက်စွဲနဲ့ လှမ်းစစ်ပါမယ်။ (ဥပမာ - daily_draws table ထဲမှာ ဒီ User ID နဲ့ ဒီနေ့ရက်စွဲနဲ့ သိမ်းထားတာ ရှိလား?)
- ရှိခဲ့ရင် (Already Drawn): အသစ်မဆွဲတော့ဘဲ Database ထဲက အဲဒီကတ်ကိုပဲ ယူပြီး မျက်နှာပြင်မှာ တန်းပြပါမယ် (Animation တော့ ထည့်ပြလို့ ရပါတယ်)။
- မရှိသေးရင် (New Draw): ကတ် ၇၈ ကတ်ထဲကနေ Random တစ်ကတ်ကို ဆွဲထုတ်မယ်။ ပြီးရင် အဲဒီကတ်ကို ဒီနေ့အတွက် Daily Draw အဖြစ် Database ထဲမှာ အလိုအလျောက် သွားသိမ်း (Auto-save) လိုက်ပါမယ်။ ပြီးမှ မျက်နှာပြင်မှာ ပြပါမယ်။

အဆင့် (၃) - Guest ဖြစ်ခဲ့လျှင် (Token မရှိလျှင်) - Pro Trick 💡
Guest တွေက အကောင့်မရှိတဲ့အတွက် Database ထဲ သွားစစ်လို့ မရပါဘူး။ ဒါပေမယ့် သူတို့လည်း Refresh လုပ်တိုင်း ကတ်မပြောင်းသွားအောင် Browser ရဲ့ localStorage ကို သုံးပြီး ဖြေရှင်းလို့ ရပါတယ်။
- Guest ဝင်လာရင် Browser ရဲ့ localStorage ထဲမှာ ဒီနေ့ရက်စွဲနဲ့ သိမ်းထားတဲ့ ကတ် ရှိ/မရှိ စစ်ပါမယ်။
- ရှိခဲ့ရင်: အဲဒီကတ်ကိုပဲ ပြန်ပြပါမယ်။
- မရှိသေးရင်: Random ကတ်အသစ်တစ်ခု ဆွဲမယ်။ ပြီးရင် အဲဒီကတ်နာမည်နဲ့ ဒီနေ့ရက်စွဲကို localStorage ထဲမှာ ခဏ မှတ်ထားလိုက်ပါမယ်။ 
ဒါဆို Guest က Browser ကို ပိတ်ပြီး နောက် ၂ နာရီနေမှ ပြန်လာဖွင့်လည်း ဒီနေ့အတွက်တော့ အဲဒီကတ်ပဲ ဆက်ပေါ်နေမှာပါ။ (နောက်ရက် ကူးသွားမှသာ ရက်စွဲမတူတော့လို့ ကတ်အသစ် ပြန်ပေါ်ပါမယ်)။

Page ၏ အလုပ်လုပ်မည့် အဆင့်များ (User Flow)

အဆင့် (၁) - Spread ရွေးချယ်ခြင်း (The Updated Selection Room)
- Guest တွေရော၊ Member တွေပါ အားလုံးမြင်ရမယ့် မျက်နှာပြင်မှာ စုစုပေါင်း ရွေးချယ်စရာ ၅ ခု (သို့မဟုတ် ထို့ထက်ပိုပြီး) ကို ပြသထားပါမယ်။
- Free Options (Guest များ အခမဲ့ ဖတ်နိုင်မည့် ၃ မျိုး) ဒီ ၃ ခုကိုတော့ အရောင်တောက်တောက်၊ နှိပ်လို့ရတဲ့ ပုံစံလေးတွေနဲ့ ပြထားပါမယ်။
   * ၁-ကတ် ဟောစာတမ်း (Yes/No / Quick Answer): မေးခွန်းတစ်ခုအတွက် တိကျမြန်ဆန်တဲ့ အဖြေလိုချင်သူတွေအတွက်။
   * ၃-ကတ် ဟောစာတမ်း (အတိတ်၊ ပစ္စုပ္ပန်၊ အနာဂတ်): အခြေခံအကျဆုံးနဲ့ လူကြိုက်အများဆုံး ရွေးချယ်စရာ။
   * ၃-ကတ် ဟောစာတမ်း (အခြေအနေ၊ အကြံပြုချက်၊ ရလဒ် - Situation, Action, Outcome): ဒါကလက်တွေ့ ပြဿနာတစ်ခုခုကို ဘယ်လိုဖြေရှင်းရမလဲ အကြံတောင်းချင်တဲ့ User တွေ အရမ်းကြိုက်မယ့် ပုံစံလေးပါ။ 

- Premium Options (Member သီးသန့် ရွေးချယ်စရာများ) ဒီအရာတွေကိုလည်း မျက်စိရှေ့မှာ ချပြထားပါမယ်၊ ဒါပေမယ့် UI အရ နည်းနည်း ကွဲပြားနေပါမယ်။
   * Celtic Cross (၁၀-ကတ်) အသေးစိတ် ဟောစာတမ်း
   * အချစ်ရေး သီးသန့် (၅-ကတ်) ဟောစာတမ်း
   * နောက်ထပ် option  များ...

- UI Design: 
   * Guest တွေအတွက် ဒီ Premium ကတ်တွေရဲ့ ပုံပေါ်မှာ သော့ခလောက်ပုံ (Lock Icon 🔒) လေးတွေ ထောင့်နားမှာ တပ်ထားပါမယ်။ 
   * Member  တွေမှာတော့ ဖြုတ်ထားပါမယ်။

** Logic: Guest က သော့ခတ်ထားတဲ့ Spread ကို သွားနှိပ်လိုက်ရင်၊ ချက်ချင်းကြီး Page မပြောင်းသွားဘဲ "ဒီ အသေးစိတ် ဟောစာတမ်းကို ဖတ်ရှုရန် Member ဝင်ပါ" ဆိုတဲ့ လှပတဲ့ Pop-up (Modal Box) လေး အရင်တက်လာပါမယ်။ အဲဒီက "Login ဝင်ရန်" ဆိုတဲ့ ခလုတ်ကို နှိပ်လိုက်မှသာ /login Page ကို လမ်းကြောင်းလွှဲ (Redirect) ပေးလိုက်ပါမယ်။

အဆင့် (၂) - ကတ်မွှေခြင်း နှင့် ဖြန့်ခင်းခြင်း (Shuffle & Fan Out)
- Spread ရွေးပြီးတာနဲ့ မျက်နှာပြင်မှာ ကတ်ထုတ်ကြီး (Deck) တစ်ခု ပေါ်လာပါမယ်။
- Animation: "ကတ်မွှေရန် နှိပ်ပါ" ဆိုတဲ့ Button လေး ပါမယ်။ နှိပ်လိုက်ရင် CSS/JS သုံးပြီး ကတ်ထုတ်ကြီးကို တကယ် မွှေနေတဲ့ Animation လေး (ကတ်တွေ ထပ်လိုက်၊ ခွဲလိုက်) ပြပါမယ်။
- မွှေပြီးတာနဲ့ ကတ် ၇၈ ကတ်လုံးကို စားပွဲပေါ်မှာ မှောက်လျက် ယပ်တောင်ပုံစံ (Fan out) ဒါမှမဟုတ် အတန်းလိုက်လေး ဖြန့်ခင်းလိုက်ပါမယ်။

အဆင့် (၃) - ကတ်ရွေးချယ်ခြင်း (The Gathering)
- User က ဖြန့်ခင်းထားတဲ့ ကတ်တွေထဲကနေ တစ်ကတ်ချင်း Click နှိပ်ပြီး ရွေးပါမယ်။
- Animation: User က ကတ်တစ်ခုကို နှိပ်လိုက်တာနဲ့ အဲဒီကတ်လေးက ဖြည်းဖြည်းချင်း လှည့်ပြီး (Rotate)၊ Screen ရဲ့ ဘေးဘက် အပြင်ကို Fade-out ဖြစ်ပြီး ပျံထွက်သွားပါမယ်။

အဆင့် (၄) - ကတ်လှန်ခြင်း နှင့် အဖြေ Box (The Reveal & The Box)
- ကတ်တွေ ရွေးပြီးသွားတဲ့အခါ မျက်နှာပြင် အလယ်မှာ နေရာလွတ် Placeholder လေးတွေ ပေါ်လာပါမယ်။
- Animation 1: အပြင်ကို ထွက်သွားတဲ့ ကတ်တွေက တစ်ကတ်ချင်းစီ အပြင်ကနေ ပျံဝင်လာပြီး (Translate In)၊ မျက်နှာပြင် အလယ်မှာရှိတဲ့ Placeholders တွေဆီမှာ မှောက်လျက် နေရာယူပါမယ်။
- Click 1 (To Flip): User က မှောက်ထားတဲ့ ကတ်ကို နှိပ်လိုက်ရင် 3D Flip Animation နဲ့ လှန်သွားပါမယ်။
- Click 2 (To Read): လှန်ထားတဲ့ ကတ်ကို User က ထပ်ပြီး နှိပ်လိုက်ရင် -
	* မျက်နှာပြင်အနှံ့မှာ မှောင်သွားတဲ့ နောက်ခံ (Semi-transparent Overlay/Backdrop) တစ်ခု ပေါ်လာမယ်။
	* အလယ်တည့်တည့်မှာ လှပတဲ့ Box (Pop-up Box) တစ်ခု Fade-in ဖြစ်ပြီး ပေါ်လာပါမယ်။ အဲဒီ Box ထဲမှာမှ ကတ်ရဲ့ နာမည်၊ Keywords နဲ့ အသေးစိတ် ဟောစာတမ်းအပြည့်အစုံကို သပ်သပ်ရပ်ရပ် ပြသပါမယ်။ (Box မှာ Close ခလုတ် ပါပါမယ်)။

အောက်ဆုံး Action Area (သိမ်းဆည်းခြင်း)
Box ကို ပိတ်လိုက်ပြီး ဟောစာတမ်း အကုန်ဖတ်ပြီးသွားတဲ့အခါ လုပ်စရာအတွက် အောက်ခြေမှာ ခလုတ်လေးတွေ ပြထားမယ်။
